<p className="eyebrow">Overview</p>

holaOS is a desktop app where each **workspace** is a long-running AI environment — an agent that carries memory across sessions, platform apps it can use, and the files, outputs, and automations it builds up over time.

It's built for work that doesn't fit in a single prompt. Drafting a week of LinkedIn content around a product launch, triaging a busy inbox every morning, pulling a weekly research digest from a handful of sources — these are tasks that need continuity, not just a capable model. holaOS gives that continuity a structure.

## The mental model

**Workspace** — the unit of organization. Each workspace is self-contained: its own agent identity, its own memory, its own installed apps and skills. You'd create one workspace for your LinkedIn presence, a separate one for inbox triage, and another for competitive research. They don't share state.

**Agent** — the entity doing the work inside a workspace. Between sessions, the agent retains memory of prior runs, past decisions, and accumulated context. Its standing behavior is defined by an `AGENTS.md` file in the workspace — you can read and edit that file directly. The agent reads it at the start of every run.

**Apps and Skills** — how the agent's capabilities are extended. Apps are platform integrations — LinkedIn, Gmail, Reddit, GitHub, Sheets, and others — that give the agent real actions on real platforms. Skills are markdown instruction packs (`SKILL.md` files) the agent loads when relevant, encoding reusable patterns like how to write in your voice or how to structure a weekly report. You install Apps from the marketplace at `/apps`; Skills can be installed from `/skills` or authored directly inside the workspace.

Two more pieces worth knowing before you go deeper:

**Automations** let a workspace run on a schedule or in response to an event, without you typing a message. The agent wakes up, does the work, and accumulates outputs over time.

**Templates** are workspaces others have built, iterated on, and published. Instead of starting from blank, you fork a template and adapt it — the agent identity, installed apps, and standing instructions come pre-configured.

## Built on holaOS (open-source)

holaOS Desktop runs on top of `holaOS`, an open-source environment layer that defines the workspace contract, memory surfaces, runtime harness, and capability boundary that make long-horizon work durable. The whole platform — runtime, harness, and workspace structure — is inspectable and open to contribution. For the full argument behind this design, read [Environment Engineering](/docs/concepts/environment-engineering).
