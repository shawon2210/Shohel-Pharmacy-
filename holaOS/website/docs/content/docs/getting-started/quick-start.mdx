Install holaOS Desktop, open a workspace, connect a model provider, and send your first task. That's the whole path.

## 1. Install holaOS Desktop

<Tabs items={['Installer', 'From source']}>
<Tab value="Installer">

Grab the latest signed build from [github.com/holaboss-ai/holaOS-priv/releases](https://github.com/holaboss-ai/holaOS-priv/releases). macOS (Apple Silicon) is available today; additional platforms are on the way.

Download the `.dmg` for your platform, open it, drag holaOS into your Applications folder, and launch it from there. On first open, macOS may prompt you to approve the app in **System Settings → Privacy & Security** — click **Open Anyway** and the app will launch normally from that point on.

</Tab>
<Tab value="From source">

This path is for contributors who want to modify the desktop, runtime, or harness code. Requires `git`, `node` 22+, and `npm`.

```bash
git clone https://github.com/holaboss-ai/holaOS-priv.git
cd holaOS
npm run desktop:install
cp desktop/.env.example desktop/.env
npm run desktop:prepare-runtime:local
npm run desktop:dev
```

`desktop:dev` validates the environment, stages a local runtime bundle, and launches Electron. For the full six-step walkthrough and dev loops, see [Install from Source](/docs/contribute/start-developing/from-source).

</Tab>
</Tabs>

## 2. Open your first workspace

When Desktop opens, the first step is to open or create a workspace. Three options are available on the launch screen:

- **Blank workspace** — starts with an empty agent context and no pre-installed apps. Best if you have a specific setup in mind from the start.
- **Marketplace Template** — fork a pre-built workspace from [holaos.ai/marketplace](/marketplace). The agent identity, installed apps, and standing instructions are already in place, so the workspace is ready to run from the moment you open it.
- **Local folder** — point Desktop at an existing workspace directory already on your machine.

For a first run, a marketplace Template is the fastest path to a meaningful result. Because the agent, apps, and context come pre-configured, the first task you send will have something concrete to work with rather than starting from nothing.

Each workspace is self-contained — its own agent identity, memory, installed apps, and standing instructions are isolated from every other workspace on the machine.

## 3. Configure your model provider

Model configuration happens inside the workspace after it is open, not during installation. Once a workspace is loaded, open the model settings panel — accessible from the settings icon or the model indicator in the workspace toolbar — and select a provider.

Four providers are supported:

- **OpenAI** — GPT-series models. Requires an OpenAI API key.
- **Anthropic** — Claude models. Requires an Anthropic API key.
- **OpenRouter** — Routes across models from many providers under a single key. Useful if you want to switch models or manage spend across providers.
- **Ollama** — Runs models locally on your machine. No API key needed; Ollama must be installed and running before selecting this option.

For cloud providers, paste your API key into the credentials field and select the model you want to use. For Ollama, point the provider at your local instance and pick from the models you have already pulled. Provider and model selection can be changed at any time, and the change applies only to the current workspace.

## 4. Send your first task

Type your task into the message pane and send it. Specific instructions produce better results than vague ones — for example: "Draft three LinkedIn posts about our Q2 product launch, each under 200 words, with a different angle per post."

The agent reads the workspace's standing instructions, plans the work, and begins executing. If apps are installed in the workspace, the agent may call them to pull in context or stage outputs directly. Results appear in the workspace as the run progresses.

By default, operations that have a public or financial side-effect — publishing a post, sending a message, triggering a paid action — pause and ask for your approval before anything is submitted. You review the proposed action, approve or redirect it, and the agent continues from there.
